# Мобильный хоспис — автоматизация тестирования

## Описание проекта
Проект содержит автоматизированные UI-тесты для мобильного приложения «Мобильный хоспис».  
Тесты написаны на **Java** с использованием фреймворка **Espresso**, отчетность формируется в **Allure**.

**Ссылка на проект:** [https://github.com/AVMihel/Diplom_QA](https://github.com/AVMihel/Diplom_QA)

## Технологический стек
| Компонент       | Технология         |
|-----------------|--------------------|
| Язык приложения | Kotlin             |
| Язык автотестов | Java               |
| UI-тестирование | Espresso           |
| Запуск тестов   | AndroidJUnitRunner |
| Отчетность      | Allure Framework   |
| Сборка          | Gradle             |

## Требования к окружению
Для запуска автотестов необходимо:

- Android Studio (рекомендуется версия Hedgehog 2023.1.1 или новее)
- JDK 11 (устанавливается вместе с Android Studio)
- Android SDK (API 29+)
- Эмулятор Pixel 4 Android 10.0 (API 29):
  - Разрешение экрана: 1080 x 2280
  - Язык системы: Русский
- Git

## Подготовка к запуску

### 1. Клонирование репозитория
- bash
  git clone https://github.com/AVMihel/Diplom_QA.git
  cd Diplom_QA

### 2. Открытие проекта в Android Studio
- Запустите Android Studio
- Выберите **File → Open**
- Укажите путь к клонированному проекту
- Дождитесь синхронизации Gradle

### 3. Запуск эмулятора
**Через Android Studio:**
- Откройте **Device Manager**
- Создайте эмулятор **Pixel 4 API 29**, если его еще нет
- Запустите эмулятор кнопкой ▶️

## Запуск автотестов

### Запуск конкретного тестового класса
1. Выберите тестовый класс в структуре проекта слева (например, `AuthorizationTest.java`)
2. Нажмите правой кнопкой мыши → Run

### Альтернативный способ запуска
1. Откройте тестовый класс
2. Нажмите зеленый треугольник слева от имени класса или метода
3. Выберите **Run 'testClassName'**
